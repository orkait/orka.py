# Compression frontier: HybridIntelligence-0.5B (FalconH1 hybrid)

Pushing a real 0.5B hybrid **Mamba + Attention** model (`MerlinSafety/Hybrid-
Intelligence-0.5B`, FalconH1, 521M params bf16) to the highest compression that
stays usable. All runs RTX 3060, 10 GB cap. Quality = ppl ratio vs fp16 on a
held sample + generation coherence (does it stay factual: Paris, 100C, colors).

## Headline
**3 bpw uniform = 4.33x (995->230 MB) at ppl ratio 1.10, fully usable** - coherent
and factual. That is the pinnacle at *usable* quality. Below 3 bpw the model
degrades fast and neither mixed allocation nor recovery saves it (a 0.5B model has
little redundancy to exploit - small models are inherently less compressible).

## Frontier

| config | size | ratio | ppl ratio | generation |
|---|---:|---:|---:|---|
| fp16 | 995 MB | 1.0x | 1.00 | reference |
| 4 bpw uniform | 294 MB | 3.38x | 1.10 | factual |
| **3 bpw uniform** | **230 MB** | **4.33x** | **1.10** | **factual (Paris, 100C)** |
| 2.75 bpw uniform | 199 MB | 5.0x | 1.39 | borderline (colors ok, Paris lost) |
| 2.5 bpw uniform | 180 MB | 5.53x | 2.36 | broken (hallucinates) |
| 2.5 bpw + LDLQ (fixed) | 180 MB | 5.53x | 2.08 | degraded |
| 2.5 bpw + distill | 180 MB | 5.53x | 2.07 | degraded, lost facts |
| mixed (ff 2bpw, mamba 4bpw) | 210 MB | 4.7x | 2.19 | broken |
| sensitivity-split @ 2.74 avg | 199 MB | 5.0x | 1.59 | worse than uniform |

## What we learned
- **Uniform allocation is the sweet spot.** Both mixed-allocation attempts
  (crush-robust / protect-sensitive) underperformed uniform at the same average
  bpw. feed_forward is 43% of params and cannot survive 2 bpw, so there is no
  robust headroom to redistribute.
- **3 bpw == 4 bpw quality (both 1.10)** - 4 bpw was overkill; 3 bpw is free 28%.
- **Recovery does not break the floor on a 0.5B model.** distill nudged 2.5 bpw
  2.36 -> 2.07, still unusable. Small models lack the redundancy that lets larger
  models (Qwen-1.5B hit 4 bpw @ 1.16) recover at aggressive bitrates.
- **orka handled a non-standard hybrid arch out of the box** - mamba `in_proj`/
  `out_proj` quantize fine alongside attention and MLP.

## Two bugs fixed along the way
1. **Error-compensation on the output head + mamba layers** made ppl WORSE than
   plain VQ (4 bpw 1.10 -> 1.50) and caused degenerate looping generation. Block-
   OBS minimises the *linear* output error `E[(Wx-What x)^2]`, valid only when the
   layer feeds a locally-linear path. The output head (softmax) and mamba (SSM
   scan) violate that. Now skipped -> they keep plain VQ.
2. **autoquant could not read bf16 checkpoints** (`safe_open(...,"np")` raised on
   bf16). Now reads via torch + fp32 upcast.

## Levers tried to break the 3bpw floor (all ruled out)
Exhaustive search confirms 3 bpw is the floor for this 0.5B model:

| lever | result |
|---|---|
| lower uniform bpw | 2.75 = 1.39 (borderline), 2.5 = 2.36 (broken) |
| mixed allocation (2 variants) | 2.19 / 1.59 - **worse than uniform** at equal avg bpw |
| ANS / entropy-code indices | **0% gain** - indices incompressible (well-fit VQ = max entropy; codebooks only 8.9%) |
| distill recovery (codebook refine) | 2.5bpw 2.36 -> 2.07, still broken |
| QLoRA recovery (rank-32 adapters) | 2.5bpw -> 1.61, but +52MB adapters = 232MB = same size as 3bpw, worse quality |

The recovery methods improve the 2.5bpw base but never beat 3bpw on the
size/quality frontier: a 0.5B model lacks the redundancy that lets larger models
recover at aggressive bitrates (Qwen-1.5B reached 4bpw @ 1.16). The 798-token
bundled calibration set also caps recovery.

## E8 lattice vs VQ on this model (validated, NOT just smol)
The lattice path (`orka.quant.lattice`) wins big on **smol** (e8x1 @ 4.41 bpw -> 1.20,
e8x2 @ 5.51 -> 1.02; VQ can't reach near-lossless). It does NOT transfer to this hybrid.
Measured on an identical wikitext sample (32x256 tok), apples-to-apples, fp16 base ppl
13.51:

| config | q_bpw | ppl ratio | size | compress |
|---|---:|---:|---:|---:|
| **VQ 3 bpw uniform** | 3.00 | **1.345** | **240 MB** | **4.34x** |
| E8 e8x1 (0.9) | 3.24 | 1.598 | 325 MB | 3.21x |
| E8 e8x1 (0.7) | 3.59 | 1.301 | 345 MB | 3.02x |
| E8 e8x1 (0.55) | 3.92 | 1.160 | 364 MB | 2.86x |
| E8 e8x1 (0.45) | 4.21 | 1.099 | 381 MB | 2.74x |
| E8 e8x2 (0.45,0.15) | 7.01 | 1.010 | 572 MB | 1.82x |

VQ **Pareto-dominates the usable end**: lattice needs 4.21 bpw / 381 MB / 2.74x to match
the ppl (1.10) VQ delivers at 3 bpw / 240 MB / 4.34x. Lattice only wins at the near-
lossless end (e8x2, ~1.01) where it costs 1.8x compression. The smol win was standard-
transformer-specific (self_attn+mlp = ~all params); a hybrid puts 78% of params in
feed_forward+mamba where row-scaled VQ + size-aware allocation beat the lattice.
**Do not unify lattice into the VQ format for hybrids** - validation killed that lever.

(Two bugs found + fixed during this: lattice `_is_quantizable` was a `self_attn`/`mlp`
allow-list that covered only **9%** of this hybrid - feed_forward/mamba silently stayed
fp16 and `avg_bpw_quantized` was fiction; the doc's earlier "1.10" VQ ratio was an easier
sample, 1.345 on standard wikitext.)

## Structural, not name-based, layer detection
The "keep the head fp16" and "skip block-OBS on SSM" decisions are now STRUCTURAL, not
name substrings (which silently broke on `feed_forward` vs `mlp`, `mixer` vs `mamba`):
- output head = a 2-D weight whose out dim == vocab_size (`get_output_embeddings()` /
  shape), not `name in (...)`.
- recurrent/SSM = a Linear whose module owns recurrence state params (`A_log`/`dt_bias`),
  not `"mamba" in name` - so a pure-Mamba `...mixer.in_proj` is caught too.
Helpers in `orka.quant.family` (`output_head_names` / `recurrent_block_names` primary,
`is_output_head` / `is_recurrent_block` name fallbacks). Every error-comp skip is now
logged (no more silent no-ops).

## Recommendation
Ship **3 bpw uniform VQ** for this model: 4.34x at ratio ~1.1-1.35 (sample-dependent),
factual generation, ~4 min to pack - the validated pinnacle at usable quality, and it
Pareto-beats E8 lattice here. Use 2.75 bpw (5.0x) only if a ~1.4 ratio / occasional fact
loss is acceptable. Do not use LDLQ on hybrid/SSM layers (auto-skipped structurally now).
